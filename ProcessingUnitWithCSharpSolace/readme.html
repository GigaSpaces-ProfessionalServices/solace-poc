GigaSpaces XAP.NET Processing Unit Example

The Processing Unit example demonstrates how to implement a basic processing unit and the different ways of deploying it. Additional documentation is available here.

Overview

This example demonstrates the following:

  • Creating a simple data feeder as a processing unit
  • Creating a simple data processor as a processing unit
  • Deploying the processing units into the service grid
  • Deploying the processing units in a stand alone container

Structure

The example contains 4 projects:

  • Processing Unit Entities project that contains data model object
  • Processing Unit Feeder project that contains the data feeder
  • Processing Unit Processor project that contains the data processor
  • PUDebugExecuter project that runs the processing units in a console application to allow easier debugging

Processing Units

The example contains two processing units and the data object

  • DataProcessor: implements the IProcessingUnit interface and it is in charge of processing the data
  • DataFeeder: implements the IProcessingUnit interface and it is in charge of feeding the cluster with unprocessed data
  • Data: simple object that represent our data model within the cluster

Building the Example

Step 1: Browse to the example directory ($GS_HOME\Examples\ProcessingUnit).

Step 2: Double-click the Build.bat file to compile the .NET, files are created in the $GS_HOME\Examples\ProcessingUnit\release\ directory and the feeder and processor are created in each project dir under \Deployment directory. This script will also copy the all the needed files for deploying the processing units within the service grid using the gs-webui.

Opening .NET Solution with Microsoft Visual Studio

Step 1: Double-click the GigaSpaces.Examples.ProcessingUnit.Data.sln file, depending on the version of Visual Studio you are using.

Step 2: Choose Build > Build Solution menu. This will build the projects and will copy all the necessary files for deployment.

Running the Example

Step 1: The example must be compiled before running.

Step 2: Start a local GigaSpaces Agent: $ ..\..\bin\gs host run-agent --auto --gsc=3

Step 3: Deploy the DataProcessor from the webUI to deploy a data grid named DataProcessor, use these attributes:

  • Cluster Schema = Partitioned
  • Number Of Instances = 1
  • Number of Backups = 1
Press button: Deploy

Step 4: Deploy the DataFeeder from the webUI to deploy a data grid named DataFeeder, no need of further attributes.
